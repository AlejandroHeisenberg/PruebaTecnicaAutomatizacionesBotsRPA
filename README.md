\# Prueba Técnica para Desarrollador RPA - Alejandro Reyes



¡Hola! En este repositorio presento la solución a las cuatro secciones de la prueba técnica. Mi objetivo no fue solo que el código funcionara, sino que fuera fácil de leer y que estuviera bien organizado para que la revisión sea lo más fluida posible.



\##  ¿Qué vas a encontrar aquí?



He dividido la entrega en 4 carpetas principales, cada una con su respectiva solución y una carpeta de \*\*Evidencias\*\* (donde puse pantallazos de los resultados para ahorrarte tiempo).



\### 1. Automatización en UiPath

Aquí hice un bot que saca información de un Excel. Me enfoqué en que el flujo fuera claro y que los nombres de las actividades expliquen por sí mismos qué está pasando.



\### 2. Flujo en Power Automate

Este es un flujo en la nube para gestionar correos. Lo configuré para que guarde archivos en OneDrive y lleve un control en Excel. En la carpeta verás el paquete para importar y fotos de cómo queda el registro.



\### 3. Procesamiento con Python

Hice un script que lee datos de empleados (CSV) y saca estadísticas rápidas. 

\* \*\*Ojo:\*\* Usé la librería `pandas`. Si quieres correrlo, recuerda hacer un `pip install -r requirements.txt`. 

\* El resultado lo genera en un archivito JSON muy ordenado.



\### 4. Validador en C#

Es una aplicación de consola que valida textos. Lo hice dinámico, así que puedes escribir tus propias palabras y elegir qué letra quieres contar. Verás que usé funciones separadas para cada validación para que el código no sea un "espagueti".



---



\##  Notas importantes para la revisión

\- \*\*Evidencias:\*\* Si no tienes alguna herramienta instalada (como UiPath), en cada carpeta dejé capturas de pantalla de la consola y de los archivos finales generados.

\- \*\*Entornos:\*\* En Python y C# limpié las carpetas de archivos temporales (como .venv, bin o obj) para que el envío sea liviano y no se "reviente" al abrirlo.
